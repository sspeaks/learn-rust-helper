## Hint 1: Conceptual Question

Migration systems need to know where they are before moving forward. Think about the structure: how do you determine the current version? What does it mean to "migrate from version N to version M"—is it one operation or a series? How do you know which individual steps to apply? And what prevents you from accidentally trying to migrate backward?

**Spoiler threshold:** Low—asks you to understand the migration state machine before coding.
